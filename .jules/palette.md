## 2025-01-20 - Add missing form label associations and input aria-labels
**Learning:** Screen readers and other assistive technologies rely on explicit associations between form controls and their text descriptions (via the `for` attribute matching the input's `id`), or `aria-label`s when no visual text label is appropriate. Missing these significantly degrades the experience for users of assistive tech, as inputs are read out generically without context. This issue was found widespread in `index.html` across various search, chat, settings, and upload inputs.
**Action:** Ensure all `<input>` elements either have an explicit `<label for="...">` associated with them, or are given a descriptive `aria-label` when a visible label disrupts the design layout. Specifically, check that `.sr-only` classes on labels do not utilize `display: none`, as this removes them from the accessibility tree; use CSS clipping techniques instead.
